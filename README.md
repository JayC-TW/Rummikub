# 拉密 Rummikub 網頁版

以 HTML、CSS、SVG、Vanilla JavaScript、Node.js 與 WebSocket 實作的拉密遊戲，可在電腦、手機與平板瀏覽器執行。

- 單人對戰：1 位真人與 1～3 位電腦 AI。
- 多人對戰：2～4 個座位，支援真人連線及 AI 補位。
- 正式前端：GitHub Pages。
- 多人伺服器：Render Node.js Web Service。

## 一、線上版本

- 遊戲：https://jayc-tw.github.io/Rummikub/

GitHub Pages 負責網頁與操作介面；Render 負責多人房間、洗牌、發牌、私有手牌、回合、AI 動作及規則驗證。

## 二、本機啟動

### 1. 安裝套件

```bash
cd Rummikub
npm install
```

### 2. 啟動 WebSocket 伺服器

```bash
npm start
```

預設網址：

```text
HTTP health：http://localhost:10000/health
WebSocket：ws://localhost:10000/ws
```

伺服器正式部署時會使用 Render 提供的 `PORT`，並監聽 `0.0.0.0`。

### 3. 啟動前端

另開終端機：

```bash
python3 -m http.server 8000
```

開啟 http://localhost:8000/。不可直接使用 `file://` 開啟 `index.html`，因為專案使用 ES6 模組。

原有啟動器也可使用：

```bash
python3 start_server.py
python3 start_server.py 9000
```

## 三、首頁與遊戲模式

桌面版首頁採左右雙欄：

- 左側：單人對戰。
- 右側：多人對戰。

手機版會自動改為上下排列。重新開始與離開多人牌局使用頁面內確認面板，不依賴可能被手機瀏覽器阻擋的系統 `confirm()`。

## 四、單人對戰

可設定：

- 1～3 位電腦對手。
- 每位 AI 的初級／中級／高級難度。
- 每回合 30／60／120 秒或無限制。

單人牌局完全在瀏覽器內執行，歷史局數及玩家勝場保存在 `localStorage`。

## 五、多人對戰

### 建立與加入房間

建立房間時可設定：

- 玩家暱稱。
- 最多 2～4 個座位。
- 各候補座位的 AI 難度。
- 每回合 30／60／120 秒或無限制。

建立後會取得四碼房號。其他玩家輸入不同暱稱與房號即可加入。

### 開始遊戲

- 只有房主可以開始。
- 未坐滿的座位會依設定補入 AI。
- 開始後房間鎖定，不再接受新玩家。
- Render 統一洗牌、發牌並決定先手。
- 每位玩家只會收到自己的牌面；其他玩家只公開手牌張數。
- 每人初始手牌 14 張，依顏色、數字排序，百搭牌置於最後。

### 真人操作

輪到本人時可以：

- 拖曳或點選手牌。
- 在桌面空白處或破冰區建立新牌組。
- 重新排列及還原本回合草稿。
- 抽一張牌並結束回合。
- 提交桌面及手牌草稿以結束回合。

Render 會驗證是否輪到本人、牌組是否合法、牌張是否完整、桌面牌是否被收回，以及首次破冰是否達 30 點。操作通過後才向所有玩家廣播新狀態。

### 破冰操作

- 未破冰前，桌面原有牌組保持鎖定。
- 只能使用自己的手牌建立新牌組。
- 首次出牌的真牌點數合計需達 30 點。
- 百搭牌可用來補足合法牌型，但不計入破冰點數。例如 `12 + 12 + 百搭牌` 的破冰分數只有 24 點，因此不能破冰。
- 將手牌誤拖到鎖定舊牌組時，介面會改為建立新的破冰牌組。

### 離開與斷線

- 點擊「離開多人牌局」會關閉 WebSocket 並返回首頁。
- 關閉分頁、重新整理或網路中斷也會被視為離房。
- 遊戲開始後真人離房，其原座位立即由中級 AI 接手。
- 若正輪到離房玩家，接手 AI 會執行該回合並繼續遊戲。
- 目前離房後不能重新取得原座位。

### 勝負與結算

- 真人或 AI 出完手牌後，所有連線玩家會同步進入結算畫面。
- 結算顯示勝者及各玩家剩餘手牌分數。
- 「重新開始」會保留目前房號與座位，重新洗牌開始新局。
- 「結束遊戲」會關閉多人連線並退出房間。

## 六、操作方式

- **選取 → 放置**：點擊或觸控牌面後，再點擊目的地。
- **拖曳**：使用 Pointer Events，可用滑鼠或手指拖曳。
- **手牌排序**：按「重新排列」依顏色及數字排序。
- **還原本回合出牌**：恢復到本回合開始時的桌面與手牌。
- **抽牌**：抽一張並結束回合，未送出的草稿不會提交。
- **結束回合**：驗證並提交本回合牌局狀態。

## 七、遊戲規則摘要

- 104 張數字牌：紅／藍／黑／橙，1～13 各兩組。
- 2 張百搭牌，共 106 張。
- 合法順組：同色連續數字，至少 3 張，1 不接 13。
- 合法群組：相同數字、不同顏色，至少 3 張、最多 4 張。
- 每人第一次出牌需使用手牌完成至少 30 點破冰。
- 出完手牌者獲勝。
- 牌堆用完且所有玩家無法出牌時，以剩餘手牌點數判定；百搭牌計 30 點。

## 八、AI 難度

所有 AI 都使用 `js/rules.js` 的相同合法性檢查。

- **初級**：破冰時隨機選擇可行組合；破冰後較常抽牌，只進行簡單出牌。
- **中級**：破冰時優先高點數組合；破冰後嘗試加入桌面牌組並建立新牌組。
- **高級**：優先增加出牌張數，進行多輪貪婪搜尋，並盡量保留百搭牌。

多人 AI 的決策、抽牌、出牌及回合推進由 Render 執行，再同步給所有真人玩家。

## 九、專案結構

```text
Rummikub/
├── index.html                 # 首頁、大廳、遊戲與結束畫面
├── css/style.css              # 版面、RWD、牌面與拖曳效果
├── js/
│   ├── ui.js                  # DOM、SVG、拖曳及畫面切換
│   ├── game.js                # 單人狀態機及多人遠端狀態
│   ├── multiplayer.js         # WebSocket 客戶端協定
│   ├── rules.js               # 共用規則引擎
│   ├── ai.js                  # AI 決策
│   └── music.js               # 背景音樂
├── server/
│   ├── index.js               # HTTP、WebSocket 與訊息路由
│   ├── room-manager.js        # 房間、座位及斷線接手
│   └── game-session.js        # 多人牌局、發牌、回合及驗證
├── tests/                     # Node.js 自動測試
├── package.json
└── start_server.py
```

## 十、測試

```bash
npm test
```

測試涵蓋：

- 房間建立、加入、人數限制及開局鎖房。
- AI 補位、房主同步及私有手牌。
- 真人抽牌、破冰提交及回合推進。
- 百搭牌不計入多人破冰 30 點。
- AI 回合。
- 離房及斷線 AI 接手。
- 多人初始手牌排序與破冰拖放。

## 十一、部署

### GitHub Pages

前端由 `main` 分支發布至 GitHub Pages：

```text
https://jayc-tw.github.io/Rummikub/
```

### Render

建立 Node.js Web Service：

```text
Branch: main
Build Command: npm ci
Start Command: npm start
Health Check Path: /health
```

程式會依執行環境選擇 WebSocket：

```text
localhost → ws://localhost:10000/ws
```

## 十二、目前限制

- 沒有帳號、登入、排行榜或跨裝置戰績。
- 房間及牌局尚未保存到 Redis／資料庫。
- 斷線後由 AI 永久接手，目前不支援真人重新連回原座位。
- 多人回合限時由伺服器執行；真人逾時會自動抽牌並推進回合。
