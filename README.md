# 拉密 Rummikub 網頁版

純前端（HTML + CSS + SVG + Vanilla JS ES6 模組）實作的拉密桌遊，支援 1 位真人玩家與 1~3 位電腦 AI 對戰，可在電腦、手機、平板瀏覽器執行。

## 一、如何啟動

因為程式使用 ES6 模組（`<script type="module">`），瀏覽器基於安全性限制不允許直接以 `file://` 開啟 `index.html`，需透過本機伺服器啟動。

**推薦方式**：在終端機執行啟動器，啟動成功後會自動開啟瀏覽器：

```bash
cd Rummikub
python3 start_server.py          # 預設埠號 8000
python3 start_server.py 9000     # 或指定埠號
```

按 `Ctrl+C` 停止服務。若埠號被占用會自動改用下一個可用埠號。

也可以手動啟動：

```bash
python3 -m http.server 8000
# 或使用 Node: npx serve .
```

接著在瀏覽器開啟 `http://localhost:8000/`。

## 二、檔案結構

```
/Rummikub
  index.html        # 頁面結構（開始畫面／遊戲畫面／結束畫面）
  /css/style.css     # 版面、RWD、動畫、拖曳視覺效果
  /js/rules.js        # 合法牌組驗證引擎（玩家與 AI 共用）
  /js/ai.js           # 電腦 AI 決策邏輯（依難度分層）
  /js/game.js          # 遊戲狀態機（牌堆、手牌、桌面、回合流程）
  /js/ui.js            # DOM / SVG 渲染與拖曳／點選互動
```

## 三、操作方式

- **選取 → 放置**：點擊（或觸控）一張牌會反白選取，接著點擊目的地（桌面空白處＝建立新組合、既有組合＝加入該組合）即可移動。
- **拖曳**：直接以滑鼠或手指按住牌並拖曳到目的地放開（使用 Pointer Events，滑鼠與觸控行為一致）。
- **手牌排序**：拖曳手牌可調整順序；也可按「重新排列」依顏色／數字自動排序。
- **破冰前限制**：尚未破冰時，桌面上原有的牌組會標示為鎖定（灰色實線框），無法拖入或拆解，只能用手牌組成全新的牌組；新牌組總點數需達 30 點才能結束回合。
- **破冰後**：可自由拖曳桌面上所有牌，重組任意組合，只要結束回合時所有組合皆合法（綠框）即可。牌組不合法時會顯示紅框。
- **按鈕**：
  - 「重新排列」：手牌依顏色/數字排序（不影響桌面）
  - 「還原本回合出牌」：把桌面與手牌還原成本回合開始時的狀態
  - 「抽牌」：從牌堆抽 1 張牌並結束本回合（會捨棄尚未送出的桌面調整）
  - 「結束回合」：驗證桌面所有組合，通過即送出本回合的出牌

## 四、遊戲規則摘要

- 104 張數字牌（紅／藍／黑／橙，1~13 各兩組）+ 2 張百搭
- 每人 14 張手牌，隨機決定先手
- 合法牌組：同色連續數字「順組」或同數字不同色「群組」，至少 3 張，1 不接 13
- 每位玩家第一次出牌（破冰）需使用自己手牌組成新牌組（順組、群組可混用多組），真牌點數加總達 30 點；破冰可使用百搭牌，但百搭牌本身不計入 30 點
- 出完手牌者獲勝；牌堆抽完且所有人皆無法出牌時，比較剩餘手牌點數，最低者獲勝（百搭以 30 點計）
- 回合限時可選 30／60／120 秒或無限制，超時自動抽牌並跳過

## 五、AI 難度邏輯簡述（詳見 `js/ai.js` 註解）

所有難度皆透過 `rules.js` 的同一套合法性檢查函式做判斷，確保與玩家規則一致。

- **初級**：破冰時隨機嘗試手牌組合，找到任一組總點數達 30 即出牌；破冰後僅有 40% 機率主動出牌（其餘保守抽牌），且出牌時隨機挑選一組合法牌組，不會重組桌面。
- **中級**：破冰時挑選點數總和最大的組合；破冰後會嘗試把手牌插入桌面既有組合（優先出大點數牌），並找出手牌中可獨立成組的牌一併出清。
- **高級**：破冰時盡量湊出「出牌張數最多」且達標的組合，加速清空手牌；破冰後進行多輪貪婪搜尋，反覆嘗試把手牌插入桌面任一組合、或組成新牌組，插牌時優先使用一般牌而非百搭牌，藉此保留百搭牌的彈性；找不到任何出牌方式才會抽牌。

每次 AI 出牌前都會先在畫面顯示「思考中…」（模擬 500ms~1500ms 延遲，依難度略有差異），並在套用結果前用規則引擎再次驗證，避免 AI 邏輯與玩家規則不一致。

## 六、其他

- 對戰紀錄（總局數／玩家勝場）以 `localStorage` 保存，顯示於開始畫面下方。
- 遊戲邏輯（`rules.js` / `ai.js` / `game.js`）與畫面渲染（`ui.js`）分離，方便單獨測試或除錯。
